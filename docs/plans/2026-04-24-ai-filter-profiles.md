# AI Filter Profiles Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the single-prompt AI filter with named, reusable filter profiles assignable to each Search.

**Architecture:** New `ai_filter_profiles` table per user (name, prompt, per-profile blacklist, is_default). Existing `links` gets nullable `filter_profile_id`. Scanning resolves `link.filter_profile_id ?? default profile`; if neither, skip AI filtering. Global blacklist stays on `advanced_matching` and unions with the profile blacklist at filter time.

**Tech Stack:** Supabase (cloud project `rtsjqwasyzbverpkgaqm`), Electron main-process IPC, React renderer, Deno edge functions.

**Note on verification:** This repo has no automated tests. Each phase ends with a manual verification step (curl, sign-in-as-user script in `/tmp/query_as_user.cjs`, or UI smoke test). Commit after each task.

## Phase 1 — Schema

### Task 1.1: Write migration

**File:** `apps/backend/supabase/migrations/20260424000000_ai_filter_profiles.sql` (create)

```sql
create table public.ai_filter_profiles (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  user_id uuid not null default auth.uid() references auth.users(id) on delete cascade,
  name text not null,
  chatgpt_prompt text not null default '',
  blacklisted_companies text[] not null default '{}',
  is_default boolean not null default false
);
create unique index ai_filter_profiles_one_default_per_user
  on public.ai_filter_profiles (user_id) where is_default;
alter table public.ai_filter_profiles enable row level security;
create policy "users manage own profiles" on public.ai_filter_profiles
  for all to authenticated
  using (auth.uid() = user_id) with check (auth.uid() = user_id);

alter table public.links
  add column filter_profile_id bigint
  references public.ai_filter_profiles(id) on delete set null;

alter table public.advanced_matching drop column chatgpt_prompt;
```

**Verify:** Apply via Supabase dashboard SQL editor. Then `curl` with service role key: `select * from ai_filter_profiles` returns `[]`; `\d links` shows the new column. Commit.

## Phase 4 — Edge functions

### Task 4.1: `apps/backend/supabase/functions/_shared/advancedMatching.ts`

- Fetch link with `filter_profile_id`.
- Resolve profile: `select * from ai_filter_profiles where (id = :link.filter_profile_id) or (is_default and user_id = :job.user_id) order by id = :link.filter_profile_id desc limit 1`.
- If no profile found, **skip** AI exclusion (return empty exclusions).
- Fetch global blacklist from `advanced_matching.blacklisted_companies`.
- Union global + profile blacklists for company exclusion.
- Pass profile.chatgpt_prompt to the OpenAI call (current line ~64).

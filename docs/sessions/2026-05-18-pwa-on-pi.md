## Key Findings
- The Pi's `f2a-web-ui` systemd slot was reserved for `apps/serverWebUI`
  (a scaffold). Repurposing it for `apps/webapp` is cheaper than building
  serverWebUI.
- `@serwist/next` v9 doesn't support Turbopack → build flag `--webpack`.
- Next standalone output omits `.next/static` and `public/` — needed an
  explicit copy step.
- SW must never cache authenticated Supabase responses (cross-user leak).
  Offline data lives in IDB keyed by `user.id`, cleared on auth change.
- Three-layer kill switch: `?nosw=1`, compile-time `KILL_SW`, `/sw-reset`.
